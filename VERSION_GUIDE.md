# 🏛️ ACCORD Project: Versioned Directory Guide

I have organized your project into three versioned folders so you can easily track the evolution of the research and access the specific files you need.

---

## 📁 [v1_foundations](./v1_foundations/)
**Focus:** The initial math proofs and core concepts.
*   `ACCORD_Proofs.md`: Simple math explanations.
*   `ACCORD_Proofs.tex`: Formal LaTeX proofs.
*   `README.md`: Original project overview.

## 📁 [v2_rigorous_validation](./v2_rigorous_validation/)
**Focus:** The experimental data, "Elite Findings," and project audits.
*   `experimental_results.csv`: The master data from all benchmarks.
*   `ACCORD_Impact_Findings.md`: Details on the "Confidence Collapse."
*   `ACCORD_FINAL_MASTER_RECORD.md`: The complete project status audit.
*   `PROJECT_COMPLETE_REPORT.md`: The high-level executive summary.

## 📁 [v3_publication_ready](./v3_publication_ready/)
**Focus:** The final, submission-quality IEEE papers.
*   `ACCORD_IEEE_Full_Detailed_Paper.tex`: The expanded 6-page paper with ablation studies.
*   `ACCORD_IEEE_Submission_Ready.tex`: The optimized 4-page conference version.
*   `ACCORD_IEEE_Conference.tex`: The original draft paper.

---

## 🛠️ Source Code & Scripts
The core logic remains in the root folders for easier execution:
*   `src/`: The Python engine (aggregator, copula, splitter).
*   `scripts/`: The validation and research scripts.
*   `requirements.txt`: Project dependencies.

**This structure allows you to see the project's growth from a simple idea (v1) to a proven engine (v2) and finally to a world-class paper (v3).**
